Introduction

In JavaScript, an object is a form of data storage that is often used to represent real-world objects. Objects consist of properties and methods. Properties describe the object itself, and methods define what the object can do.

For example, you are a person (as far as we know...), and you have a variety of characteristics that describe you, such as your:

  • Name
  • Age
  • Occupation

If we were to create an object to describe you, these characteristics would be among your properties.

You can also do stuff, like introduce yourself. In the object about you, a function to introduce yourself would be among your methods.

Creating an Object

The simplest way to create an object, is to declare a variable, give it a name, and set it equal to a pair of curly brackets with nothing in between, like this:

              
  var emptyObject = {};

However, an object like this isn't helpful, and pretty boring, so let's create an object that's a little more exciting. Perhaps a person! Since, I don't know you, let's use a fictional character like Luke Skywalker as our example:

              
  var lukeSkywalker = {
    name: 'Luke Skywalker',
    age: 19,
    occupation: 'moisture farmer',
    greeting: function() {
      console.log(Hello! My name is Luke Skywalker and I am a Moisture Farmer.");
    }
  };

See? We declared a variable and set it equal to a list of key-value pairs.

Accessing Object Data

So now we have our object, let's access the data!

We can access the data in two different ways:

  1. Dot Notation
  2. Bracket Notation

Our Luke Skywalker object has been saved in a script on this web page. Open up your browser's development tools, and go to the JavaScript console if you want to practice accessing the data.

In the console, type the following commands, pressing Enter/Return after each one:

              
  //Dot notation
  lukeSkywalker.age
  lukeSkywalker.occupation
  lukeSkywalker.greeting()

  //Bracket notation
  lukeSkywalker['age']
  lukeSkywalker['occupation']
  lukeSkywalker['greeting']()

Cool beans! You now know how to access object data.

Setting Object Data

If simply accessing object wasn't enough for you, then you're in luck! You can also define new properties or change the value of existing properies by declaring the properity you want to set (using dot or bracket notation), like this:

              
  lukeSkywalker.age = 25;
  lukeSkywalker['occupation'] = 'Jedi Knight';

Now type the same commands as in the previous section, and the new values should appear!

              
  lukeSkywalker.age
  lukeSkywalker.occupation
  lukeSkywalker.greeting()

Woah! Pretty cool right?

Object Constructors

"THIS IS AMAZING! NOW I WANT TO MAKE A HAN SOLO OBJECT!" - you, probably...

OK! Fine! Relax! I'll show you how!

To create Han Solo, we could type everything out, like this:

              
  var hanSolo = {
    name: 'Han Solo',
    age: 29,
    occupation: 'smuggler',
    greeting: function() {
      console.log("Hello! My name is Han Solo and I am a smuggler.");
    }
  };

But that take too long! You're a busy person. You have lots objects you want to create a not enough time to do so!

Fret not, young Padawan, we can speed up the process of object creation by declaring a function that acts as an object constructor. For example:

              
  function Person(name, age, occupation) {
    this.name = name;
    this.age = age;
    this.occupation = occupation;
    this.greeting = function() {
      console.log(`Hello! My name is ${this.name} and I am a ${this.occupation}.`);
    }
  }

The code above will create a new object with all of the properties and methods you need. Create your Han Solo object by going to your console and typing this:

              
                var hanSolo = new Person('Han Solo', 29, 'smuggler');

Then access the data like before:

              
  hanSolo.name
  hanSolo.age
  hanSolo.greeting()

Look at that! You're getting pretty good at this!

JSON

JSON (short for "JavaScript Object Notion") is a human-friendly way of structuring data to be transmitted from one place to another. The syntax is based on JavaScript objects!

Here is an example of what it looks like:

              
  {
    "organizationName": "Jedi Order",
    "headquarters": "Jedi Temple",
    "headquartersLocation": "Coruscant",
    "members": [
      {
        "name": "Yoda",
        "age": 900,
        "lightsaberColor": "green"
      },
      {
        "name": "Obi-Wan Kenobi",
        "age": 30,
        "lightsaberColor": "blue"
      },
    ]
  }

It kinda looks a lot like an object, right? That's the point!

When you start building your own web applications, you will likely need to request data from some other source. When you do so, it will probably be delivered to you in JSON format.

What you can do is save that JSON data as a variable in your source code, and access it the same way you would any other object!
